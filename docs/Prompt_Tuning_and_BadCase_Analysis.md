# 🛠️ MediVision Agent: Prompt 调优与 Bad Case 分析记录

## 1. 背景与痛点
在早期版本的多模态 Agent 测试中，我发现大语言模型（LLM）在接收底层计算机视觉（CV）模型返回的 JSON 诊断结果时，偶尔会出现以下“幻觉”（Hallucination）或信息遗漏的 Bad Case：
- **无视低置信度**：即使 CV 返回的 confidence score 仅有 0.55，大模型仍可能用极其肯定的语气告知患者“您确诊了某某病”。
- **遗漏视觉证据**：工具已经成功生成了 Grad-CAM 热力图，并在 JSON 中返回了 `heatmap_path`，但 LLM 在最终生成的报告中忘记使用 Markdown 语法渲染该图片。
- **警报感知迟钝**：面对恶性黑色素瘤（MEL）等高危结果，LLM 的回复过于平淡，缺乏临床急救和活检的强烈引导。

## 2. 核心解决方案：CoT (思维链) + Few-Shot (少样本提示)
为了解决上述痛点，我在 `llm_engine.py` 的 System Prompt 中引入了严格的 **ReAct 约束** 和 **<thinking> 内部思维链机制**。

### 优化的 System Prompt 节选：
```markdown
面对患者的询问，你必须遵循以下步骤：
1. 必须主动调用 `analyze_skin_lesion` 工具分析患者提供的图片路径。
2. 拿到工具返回的 JSON 数据后，你必须首先在内部进行思考，思考过程用 <thinking> 标签包裹。在思考阶段，你必须核对：模型预测类别、置信度、热力图路径、是否有临床高危警告。
3. 思考完成后，输出给患者的最终报告。报告中必须使用 Markdown 格式嵌入生成的热力图。
```

## 4. 结论
通过 `<thinking>` 标签的引入，强制 LLM 在输出最终结论前进行“自我盘点”，彻底消除了多模态数据合并时的幻觉问题，并完美承接了底层模型的临床警告，达到了医疗级应用的安全标准。
